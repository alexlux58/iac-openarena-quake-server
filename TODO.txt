================================================================================
OpenArena AWS - Future Security & Production Improvements
================================================================================

These improvements will transform the project into a "Secure Production Blueprint"
that demonstrates enterprise-grade security practices recognized by recruiters
and hiring managers.

================================================================================
A. Eliminate SSH as the Default Access Path
================================================================================

Replace inbound SSH with AWS Systems Manager Session Manager (SSM).

Security Impact:
  - No open port 22
  - No exposed SSH keys
  - Full session auditing

Deliverables:
  [ ] Terraform: Attach AmazonSSMManagedInstanceCore to the instance role
  [ ] Security Group: Remove SSH ingress rule
  [ ] README: Document "Access via aws ssm start-session"
  [ ] Update deploy scripts to use SSM instead of SSH
  [ ] Update Ansible to use SSM plugin instead of SSH

================================================================================
B. Harden the OS and Document It
================================================================================

Add a "hardening" role or cloud-init that:
  - Enables unattended security updates
  - Configures UFW (or nftables)
  - Locks down services, disables password auth
  - Installs auditd (optional)

Deliverables:
  [ ] Create ansible/roles/hardening/ with OS hardening tasks
  [ ] Create docs/hardening.md with exact changes documented
  [ ] CI check: Verify CIS-ish settings (even if partial)
  [ ] Add hardening role to site.yml playbook
  [ ] Document all security configurations applied

================================================================================
C. Add Policy-as-Code Gates for IaC
================================================================================

In CI, enforce:
  - Terraform formatting + validate (already done)
  - IaC security scan: tfsec or checkov
  - Secret scanning: gitleaks
  - License/vuln scanning: trivy fs + osv-scanner

Deliverables:
  [ ] Update .github/workflows/ci.yml with security scanning stages:
      [ ] Add tfsec or checkov for Terraform security scanning
      [ ] Add gitleaks for secret detection
      [ ] Add trivy filesystem scan
      [ ] Add osv-scanner for dependency vulnerability scanning
  [ ] Create .pre-commit-config.yaml to catch issues before CI
  [ ] Document security scanning results and remediation process

================================================================================
D. Supply Chain and Artifact Integrity (Big DevSecOps Signal)
================================================================================

Generate an SBOM for your build artifacts (even if it's just config bundles)
Sign artifacts (or container images) with cosign
Produce provenance (SLSA-lite)

Deliverables:
  [ ] Create Makefile targets:
      [ ] make sbom - Generate SBOM for artifacts
      [ ] make sign - Sign artifacts with cosign
      [ ] make provenance - Generate SLSA-lite provenance
  [ ] Create docs/supply-chain.md explaining the flow
  [ ] Integrate SBOM generation into CI/CD pipeline
  [ ] Document artifact signing process
  [ ] Store signed artifacts in secure location
